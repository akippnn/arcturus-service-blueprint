# Blueprint compatibility

The blueprint and host control plane are versioned independently from the manifest API.

## Current generation

- Blueprint: `0.99.0-rc.2`
- Minimum Arcturus host: `0.99.0-rc.2`
- Manifest API: `arcturus.u128.org/v2`
- Project API: `arcturus.u128.org/project/v1`
- Required host capabilities: `authenticated-preflight`, `legacy-compose-handoff`

The generated `.arcturus/lock.env` records these requirements. Deployment preflight rejects an older host before image builds begin.

## Updating existing projects

Projects generated by the project-aware RC1 setup can be adopted by the replayable updater using their existing `.arcturus/project.env`. Projects from the earlier webhook, Compose, or Terraform generation do not contain enough durable setup intent for blind replay; bootstrap them once with explicit service, image, routing, health, CI, and project-spec values.

The updater preserves locally modified generated files and stages proposed replacements under `.arcturus/updates/`.
